**这是什么类型的PR？**

必须是以下之一：

/kind fix (缺陷修复)
/kind feat (新增功能)
/kind perf (性能提升)
/kind style (代码可读性提升)
/kind chore (非关键代码修改)
/kind revert (代码回退)
/kind refactor (代码重构)
/kind ci (CI配置和脚本修改)
/kind test (添加或修改测试用例)
/kind docs (添加或修改项目文档)
/kind build (编译或依赖包的修改)

## Mermaid 图注意事项

GitCode 的 mermaid 版本限制：

**禁用语法**：
- `note for` / `Note over` — 均不支持，用 `<<stereotype>>` 替代（类图）或在 alt 分支内用消息替代（时序图）
- `::` 在 participant 名称中 — 解析器误认为命名空间限定符，改为空格分隔
- `()` 在 alt/else 标签中 — 解析器误认为函数调用，改为 `-` 分隔

**消息文本限制**：
- 禁止 `()` `{}` `<>` `[]` 等括号类字符
- 禁止 em dash `—` 等 Unicode 特殊字符，用 `-` 或 `:` 替代
- 消息尽量简短英文

**箭头语法**：
- 统一用 `->>` 实线箭头，`-->>` 虚线箭头在旧版 mermaid 可能渲染异常
- 类图关系用 `--|>` (继承) 和 `..|>` (实现)
- 类图中禁止泛型参数，用文字描述替代
